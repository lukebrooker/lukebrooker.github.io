Design Systems

Break the interface down into components

SMACSS/OOCSS

Sketch your components

  • What is their purpose?
  • Which patterns currently work for users? If there isn't anything great, break patterns.
  • Work out how they will adapt as their context changes, but keep the same functionality.

e.g. Navigation

Keep the functionality/meaning consistent

Typography

rhythm(1) = 1.5em

Base all measurements off vertical rhythm


margin-bottom: rhythm(.5, $button-font-size);
padding: rhythm(.25, $button-font-size) rhythm(.5, $button-font-size);
